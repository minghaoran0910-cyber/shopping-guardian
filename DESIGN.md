# Design System

## Direction

用户在白天或夜间、刚从购物页面切换过来，情绪可能仍被促销刺激。购物守护者应像一张安静、清楚的工作台：中性表面、稳定排版、有限颜色和熟悉控件，让用户迅速进入判断状态。

设计参数：视觉变化 5/10，动效 3/10，信息密度 5/10。产品界面采用 restrained 色彩策略，强调色只用于主操作、当前选择和有意义的状态。

## Color

以苔绿色作为品牌锚点，避免电商红、金融蓝和 AI 紫。Flutter 中使用 Material 3 语义色角色，不在页面组件内散落硬编码颜色。

## Typography

- 使用系统字体栈，macOS/iOS 使用 SF Pro，Android 使用 Roboto；不下载第三方字体。
- Material 3 type roles 为基础，三端只调整平台所需的字重和行高。
- 页面标题 28/34、区块标题 20/26、正文 16/24、辅助文字 14/20、标签 13/18。
- 金额和倒计时启用等宽数字，避免内容变化导致跳动。
- 正文阅读宽度限制在约 65 个字符。

## Shape

- 大型内容区与决策卡：14px；
- 输入框、列表项和普通按钮：10px；
- 标签与筛选项可使用全圆角；
- 不使用 24px 以上的大面积圆角；
- 卡片优先使用边界或表面层级，不同时叠加宽阴影。

## Spacing

采用 4/8dp 体系：4、8、12、16、24、32、48、64。

- 手机水平边距 16；
- 平板水平边距 24；
- 桌面内容边距 32；
- 相关控件间距 8–12；
- 区块间距 24–32；
- 触控目标至少 48×48dp。

## Navigation

- macOS 宽窗口使用 NavigationRail/侧栏和内容区；
- Android 紧凑宽度使用最多 5 项底部导航，展开宽度切换 NavigationRail；
- iOS 后续适配系统 Tab Bar 与返回手势；
- “新建分析”是全局主操作，每个页面只保留一个主 CTA。

## Components

- 使用 Material 3 组件作为跨端基础，按平台适配导航、弹层、快捷键和系统手势；
- 所有表单标签常驻于输入框上方，不用 placeholder 代替标签；
- 错误信息就近显示并提供恢复动作；
- 复杂配置渐进披露；
- 加载超过 300ms 显示匹配最终结构的骨架；
- 空状态解释如何产生第一条记录，而非只写“暂无数据”；
- 决策状态同时使用图标、文字和颜色；
- 图标统一使用 Material Symbols，不使用 Emoji 充当结构图标。

## Motion

- 常规状态过渡 180–220ms；
- 只为操作反馈、状态变化和层级切换添加动效；
- 不做页面入场编排、漂浮动画和装饰循环；
- 尊重系统 Remove Animations/Reduce Motion，降级为淡入淡出或直接切换。

## Responsive Behavior

- `<600dp`：单列内容，底部导航，底部固定主操作时预留安全区；
- `600–1023dp`：NavigationRail，可使用主次双列；
- `>=1024dp`：固定侧栏，正文最大宽度约 1120，表单阅读列最大约 720；
- 多列布局在手机明确折叠为单列，不产生水平滚动；
- 所有固定区域处理系统安全区和软键盘遮挡。
